## Описание проекта

TCP Math Server — учебный проект, реализующий полноценное клиент-серверное приложение. Пользователь регистрируется (с подтверждением email), входит в систему, решает математические задачи и видит свою статистику. Сервер численно вычисляет правильные ответы и проверяет введённые пользователем значения.

### Компоненты

```
EchoServer/   — TCP-сервер (Qt Console, Docker)
Client/       — GUI-клиент (Qt Widgets)
Tests/        — Unit-тесты (Qt Test)
docs/         — Doxygen-документация, UML, стратегия тестирования
```

## Архитектура

### Общая схема

```
┌────────────────────┐   TCP :33333    ┌─────────────────────────────────┐
│  Client (Qt GUI)   │ ◄─────────────► │  EchoServer (Qt Console)        │
│                    │                 │                                 │
│  LoginWindow       │                 │  MyTcpServer  ←  View           │
│  RegisterWindow    │                 │       ↓                         │
│  TaskWindow        │                 │  functionsforserver  ← Controller│
│  GraphWindow       │                 │       ↓                         │
│  ──────────────    │                 │  DataBase (SQLite)  ← Model     │
│  MainWindow (MVC)  │                 │       ↓                         │
│  TcpClient (Sing.) │                 │  EmailSender → MailHog :1025   │
└────────────────────┘                 └─────────────────────────────────┘
```

### Паттерны

**MVC** применён на обеих сторонах:

| Слой | Сервер | Клиент |
|---|---|---|
| View | `MyTcpServer` | `LoginWindow`, `RegisterWindow`, `TaskWindow`, `GraphWindow` |
| Controller | `functionsforserver` | `MainWindow` |
| Model | `DataBase` | `TcpClient` |

**Singleton** — для объектов с единственным экземпляром:
- `DataBase` — одно соединение с SQLite на весь процесс
- `TcpClient` — одно TCP-соединение в клиенте, доступное из любого окна

**Signal–Slot** — механизм Qt для слабосвязанного взаимодействия между окнами и сетевым слоем.

## Математическая задача

Найти все корни уравнения **f(x) = 0** для кусочной функции:

```
       ⎧  a · ln(x + 1),     -1 < x < 0
f(x) = ⎨  x³ - 3x + 1,       0 ≤ x ≤ 2
       ⎩  1/(x - 2) - 1,      x > 2
```

При параметрах по умолчанию (a = 1):

| Корень | Отрезок | Значение |
|---|---|---|
| x₁ | [0.0, 0.7] | ≈ 0.347 |
| x₂ | [0.7, 2.0] | ≈ 1.532 |
| x₃ | (2.0, +∞) | = 3.0 |

### Метод бисекции

Корни ищутся функцией `bisect(a, b, eps)` — алгоритм деления отрезка пополам с точностью ε = 10⁻⁶:

1. Проверить смену знака на концах отрезка [a, b]
2. Взять середину m = (a + b) / 2
3. Если f(m) = 0 или (b − a) / 2 < ε — вернуть m
4. Иначе сузить отрезок к половине со сменой знака, повторить

Поиск всех корней (`findRoots`) перебирает интервал с шагом 0.01 и запускает `bisect` на каждом найденном участке смены знака.

## Протокол обмена

Все сообщения — текстовые строки в формате `команда&параметр1&параметр2`, заканчивающиеся символом `\n`.

### Команды

| Запрос клиента | Успех | Ошибка | Описание |
|---|---|---|---|
| `auth&login&password` | `auth+&login` | `auth-` | Авторизация |
| `reg&login&password&email` | `reg+&login` | `reg-` | Регистрация |
| `confirm&login&code` | `confirm+` | `confirm-` | Подтверждение email |
| `stat&login` | `stat&N$M&P` | — | Статистика |
| `check&task&variant&answer` | `check+` | `check-` | Проверка ответа |
| `getparams` | `params&a1&b1&...` | — | Параметры функции |
| `setparams&a1&b1&...` | `params_set+` | `params_set-` | Изменить параметры |

**stat&N$M&P** расшифровывается как:  
`N` — правильных ответов, `M` — всего попыток, `P` — очков (7 за каждый верный ответ)

### Пример диалога

```
Клиент → reg&ivan&pass123&ivan@test.com\n
Сервер → reg+&ivan\n

Клиент → confirm&ivan&483921\n
Сервер → confirm+\n

Клиент → auth&ivan&pass123\n
Сервер → auth+&ivan\n

Клиент → check&1&1&0.347\n
Сервер → check+\n

Клиент → stat&ivan\n
Сервер → stat&1$1&7\n
```

## База данных

SQLite-файл хранится в `/app/data/server.db` внутри Docker volume — данные не теряются при перезапуске контейнера.

### Таблица `Users`

| Поле | Тип | Описание |
|---|---|---|
| `login` | VARCHAR(50) PK | Уникальный логин |
| `password` | VARCHAR(64) | SHA-256 хэш пароля |
| `email` | VARCHAR(100) | Email пользователя |
| `confirmed` | INTEGER | 0 = не подтверждён, 1 = подтверждён |
| `conf_code` | VARCHAR(10) | 6-значный код подтверждения |

### Таблица `Stats`

| Поле | Тип | Описание |
|---|---|---|
| `login` | VARCHAR(50) | Логин пользователя |
| `task` | INTEGER | Номер задачи |
| `correct` | INTEGER | 1 = верно, 0 = неверно |
| `points` | INTEGER | Начисленные очки |

> Пароли **никогда не хранятся в открытом виде** — только SHA-256 хэш.

## Регистрация и Email

Поток подтверждения email:

```
Клиент                    Сервер                   MailHog
   │                         │                        │
   │── reg&ivan&pass&m@t.com→│                        │
   │                    registerUser()                 │
   │                    генерация кода 6 цифр          │
   │                    сохранение в БД                │
   │                         │── SMTP :1025 ──────────►│
   │                         │   письмо с кодом        │
   │◄── reg+&ivan ───────────│                        │
   │                         │                        │
   │  (пользователь видит код в http://localhost:8025) │
   │                         │                        │
   │── confirm&ivan&483921 ──►│                        │
   │                    проверка кода в БД             │
   │                    confirmed = 1                  │
   │◄── confirm+ ────────────│                        │
```

**MailHog** — инструмент для разработки: перехватывает все исходящие письма и показывает их в веб-интерфейсе на `http://localhost:8025`. Письма **не доставляются** реальным адресатам.

## Docker-окружение

### Сервисы

| Сервис | Образ | Порты | Назначение |
|---|---|---|---|
| `server` | собирается из `EchoServer/Dockerfile` | 33333 | TCP Math Server |
| `mailhog` | `mailhog/mailhog:latest` | 1025 (SMTP), 8025 (UI) | Тестовый SMTP |

### Сеть

Оба контейнера подключены к сети `app_network` (bridge). Благодаря этому сервер может обращаться к MailHog по DNS-имени `mailhog:1025` без явного IP.

### Volume

`db_data:/app/data` — именованный volume для хранения:
- `server.db` — база данных SQLite
- `pending_*.txt` — временные файлы при регистрации

Volume сохраняется между перезапусками контейнера. Для полного сброса: `docker-compose down -v`.
